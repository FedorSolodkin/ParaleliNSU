# Третий урок: Многопоточное программирование на C++

## Описание проекта

Проект содержит два задания по многопоточному программированию:
1. **Задание 1**: Многопоточное умножение матрицы на вектор с анализом масштабируемости
2. **Задание 2**: Клиент-серверное приложение с очередью задач

## Структура проекта

```
thirdlesson/
├── Makefile                    # Файл сборки
├── task1_matrix_vector.cpp     # Задание 1: умножение матрицы на вектор
├── task2_client_server.cpp     # Задание 2: клиент-сервер
└── README.md                   # Этот файл
```

## Требования

- Компилятор: g++ с поддержкой C++20
- Библиотеки: pthread
- Опционально: gnuplot (для построения графиков), ImageMagick или pandoc (для создания PDF)

### Шаг 4: Запуск Задания 1 (умножение матрицы на вектор)

Для матрицы размера 20000x20000:
```bash
./task1_matrix_vector 20000
```

Для матрицы размера 40000x40000:
```bash
./task1_matrix_vector 40000
```

**Что делает программа:**
- Тестирует производительность с 1, 2, 4, 7, 8, 16, 20, 40 потоками
- Измеряет время инициализации и вычислений
- Вычисляет ускорение и эффективность
- Генерирует отчеты:
  - `report_data.csv` - данные в формате CSV
  - `plot_script.gnuplot` - скрипт для построения графика
  - `detailed_report.txt` - подробный текстовый отчет с анализом
  - `speedup_chart.png` - график ускорения (если установлен gnuplot)
  - `report.pdf` - итоговый PDF отчет (если установлены необходимые утилиты)

### Шаг 5: Запуск Задания 2 (клиент-сервер)

С параметром по умолчанию (100 задач на клиента):
```bash
./task2_client_server
```

С указанием количества задач (от 5 до 10000):
```bash
./task2_client_server 500
```

**Что делает программа:**
- Запускает сервер обработки задач
- Создает 3 клиентских потока:
  - Client-Sin: вычисление синуса
  - Client-Sqrt: вычисление квадратного корня
  - Client-Pow: возведение в степень
- Каждый клиент добавляет N задач и сохраняет результаты в файл
- Автоматически проверяет корректность результатов

**Выходные файлы:**
- `results_sin.txt` - результаты вычисления синуса
- `results_sqrt.txt` - результаты вычисления квадратного корня
- `results_pow.txt` - результаты возведения в степень

### Шаг 6: Создание PDF отчета (если не создан автоматически)

Если gnuplot и ImageMagick установлены:
```bash
gnuplot plot_script.gnuplot
convert detailed_report.txt speedup_chart.png report.pdf
```

Или с помощью pandoc:
```bash
pandoc detailed_report.txt --pdf-engine=wkhtmltopdf -o report.pdf
```

Вручную можно объединить текстовый отчет и график в любой программе для работы с PDF.

## Формат выходных данных

### Задание 1

**detailed_report.txt** содержит:
- Таблицу результатов для каждого количества потоков
- Подробный анализ по группам (1-4, 7-8, 16-40 потоков)
- Оценку эффективности для каждого теста (ОТЛИЧНО/ХОРОШО/УДОВЛЕТВОРИТЕЛЬНО/ПЛОХО)
- Общий вывод с рекомендациями
- Четкое указание, с какого количества потоков результат хороший

**График** показывает:
- Линию идеального ускорения
- Фактическое ускорение для каждого количества потоков

### Задание 2

Файлы результатов содержат:
- ID задачи
- Аргументы функции
- Результат вычисления сервером
- Ожидаемый результат
- Статус выполнения (OK/FAIL)
- Итоговую статистику

## Примечания

1. Для больших матриц (40000x40000) требуется значительный объем памяти (~12 ГБ для double)
2. Результаты могут отличаться в зависимости от количества ядер процессора сервера
3. Для получения точных результатов рекомендуется запускать тесты несколько раз
4. Все вычисления проводятся на сервере согласно требованиям задания
