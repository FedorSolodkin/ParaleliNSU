# SinArray

Проект демонстрирует работу с массивами вещественных чисел (`float` / `double`) на C++. Программа заполняет массив из $10^7$ элементов значениями синуса (один полный период), вычисляет сумму элементов и выводит результат в терминал.

Сборка проекта осуществляется с помощью **CMake**. Предусмотрена возможность выбора типа данных для массива во время конфигурации сборки.

## Требования

- CMake версии 3.10 или выше
- Компилятор C++ с поддержкой стандарта C++17 (GCC, Clang, MSVC)
- Git (для клонирования репозитория)

## Сборка и запуск

### 1. Клонирование репозитория

```bash
git clone <ссылка_на_репозиторий>
cd SinArray
```

### 2. Создание директории сборки

```bash
mkdir build
cd build
```

### 3. Конфигурация проекта

По умолчанию используется тип данных **`double`**.

```bash
cmake ..
```

Для использования типа данных **`float`**, необходимо передать флаг `-DUSE_FLOAT=ON`:

```bash
cmake .. -DUSE_FLOAT=ON
```

### 4. Компиляция

```bash
make
```

### 5. Запуск

```bash
./sin_app
```

## Результаты выполнения

Теоретическая сумма синусов на полном периоде равна **0**. Однако из-за погрешностей вычислений с плавающей запятой результат отличается от нуля. Разница в точности между `float` и `double` хорошо видна на большом количестве итераций ($10^7$).

### Вариант с double (по умолчанию)

Высокая точность (около 15-17 десятичных знаков). Погрешность минимальна.

```bash
cmake ..
make
./sin_app
```

**Вывод:**
```text
-2.32831e-10
```

### Вариант с float

Пониженная точность (около 6-7 десятичных знаков). Накопленная ошибка вычислений более заметна.

```bash
cmake .. -DUSE_FLOAT=ON
make
./sin_app
```

**Вывод:**
```text
0.004531
```

> **Примечание:** Конкретные значения могут незначительно отличаться в зависимости от архитектуры процессора, компилятора и версии библиотеки `libm`, но порядок величин погрешности останется схожим.
